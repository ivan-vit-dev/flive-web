Logo concept direction: The mark should communicate live sports broadcasting — consider combining a football with a signal/broadcast icon, a play button with a pitch line, or a bold "F" in a rounded rectangle with a live pulse dot. Geometric, bold, minimal. The wordmark should match the system font — bold Inter or a similar sans-serif geometric. The violet gradient (#4c1d95 → #7c3aed → #a78bfa) is the identity colour. Amber (#fbbf24) appears only on live indicators, scores, and the clock — never in the logo itself.

Maskable icon safe zone

For icon-192.png and icon-512.png (marked purpose: "any maskable"), the visible mark must stay within the centre 80% of the canvas. Android clips icons into circles, squircles, or other shapes — content outside the safe zone gets cut.

F

Violet background fills full canvas. Mark stays within the dashed safe zone.

Recommended canvas spec

  • Background: #4c1d95 (violet-900 — matches manifest theme_color)
  • Mark colour: White or #a78bfa on violet bg
  • Safe zone: Centre 80% — mark only, no wordmark
  • Border-radius on canvas: 0px — OS applies clipping
  • Format: PNG-24 with no transparency for maskable
  • Update manifest: Set theme_color: "#4c1d95"

Export Guidelines

SVG requirements

  • Flatten all text to paths — no font embedding
  • Remove invisible layers and artboard backgrounds
  • Optimize with SVGO — target <8kb for illustrations, <2kb for icons
  • Use viewBox not fixed width/height on root
  • Dark-mode adaptable: use currentColor for strokes/fills where possible
  • Animated SVGs: use CSS @keyframes, not SMIL (better browser support)

PNG requirements

  • Export at @2x resolution minimum (display × 2)
  • Compress with TinyPNG or Squoosh — <200kb for marketing images
  • PWA icons: PNG-24 (no transparency for maskable, transparent OK for others)
  • OG images: JPEG 90% quality acceptable to stay under 300kb
  • Screenshots: PNG — exact pixel dimensions, dark theme active

Delivery format

  • Figma file with all assets on labelled frames, named by file path
  • Zip: /svg/, /png/, /ico/ folders matching public/ structure
  • File naming: lowercase, hyphens, no spaces (icon-192.png)
  • All assets placed directly into the public/ directory tree

Testing checklist

  • Favicons: verify in Chrome, Firefox, Safari, and Edge tabs
  • PWA icons: install app on Android — check home screen icon shape
  • Apple touch: add to iOS home screen — check clarity at actual size
  • OG images: test with opengraph.xyz or Meta's Sharing Debugger
  • Empty states: trigger each state in the app to verify sizing and alignment
  • Dark mode: verify all SVGs look correct in both light and dark OS themes
Estimated design effort: P1 assets (~2–3 hours for logo + icons + favicon pack using RealFaviconGenerator + 2 empty states). P2 assets (~1 day for remaining illustrations + OG image + hero screenshot). P3 assets (~1 day). The logo and favicon pack are the true blockers — a sports-simple icon (football + signal/broadcast concept) in the violet gradient should take an experienced designer 1–2 hours to finalise.
